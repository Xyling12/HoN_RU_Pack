# 📖 Правила перевода HoN RU Pack

## Структура .str файла

Каждая строка в формате:
```
КЛЮЧ\t\t\tЗНАЧЕНИЕ
```
Разделитель — **TAB** (не пробелы). Несколько TAB'ов — нормально, для выравнивания. Главное — чтобы был хотя бы один TAB между ключом и значением.

Пустые строки и строки без TAB — игнорируются.

### Цветовые коды
В значениях встречаются `^y`, `^b`, `^r`, `^g`, `^!` и т.д. — это цвета. **Не трогать**, они управляют отображением текста в игре.

Пример:
```
Ability_Andromeda1_name    ^yАндромеда^!
```
→ имя будет жёлтым цветом в игре.

### Числовые шаблоны
`{75,110,145,180}` — значения по уровням умения. **Не трогать**, игра подставляет их автоматически.

## Стиль перевода — главные правила

### ✅ Делай так:
- Активные глаголы в описаниях: **"Наносит"**, **"Замедляет"**, **"Призывает"**
- Игровой, живой русский язык
- Краткие descriptions — помещаются в тултип
- MOBA-термины из глоссария ниже

### ❌ Не делай так:
- `"Активируйте, чтобы нанести..."` → `"Наносит..."`
- `"Выберите врага, чтобы..."` → `"Атакует врага:"`
- `"Предоставляет вам..."` → `"Даёт"` или `"Пассивно:"`
- `"Нацельтесь на союзника (или на себя)"` → `"Применяется к союзнику или себе:"`
- Имена героев — **не переводить** (Accursed так и остаётся Accursed)

## Формат описаний умений

Умения имеют несколько полей:

| Ключ | Назначение |
|---|---|
| `Ability_X_name` | **Название** — короткое, ёмкое (1-3 слова) |
| `Ability_X_description_simple` | **Краткое описание** в тултипе — главное! |
| `Ability_X_description` | Полное описание (не всегда отображается) |
| `Ability_X_description2` | Дополнительный блок (Посох Мастера и т.д.) |

Приоритет: переводи в первую очередь `_name` и `_description_simple`.

---

## Что НЕ переводить

- Имена героев: Accursed, Tundra, Magmus и т.д.
- Имена предметов в описаниях если они технические (Homecoming Stone, Post Haste — они сами являются переведёнными предметами)
- Игровые идентификаторы в `{}` скобках
- Цветовые коды `^X`
